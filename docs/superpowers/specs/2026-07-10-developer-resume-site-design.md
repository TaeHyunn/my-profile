# 조태현 개발자 이력서 사이트 설계

## 1. 목적

Vue 중심 프론트엔드 엔지니어 조태현의 실무 경험을 채용 담당자가 빠르게 이해할 수 있는 한국어 단일 페이지 사이트로 만든다. 샘물정보통신의 비공개 GitHub 저장소 3개에서 확인한 실제 기여를 근거로 사용하며, 확인되지 않은 사업 성과나 수치는 만들지 않는다.

이 사이트는 다음 목표를 가진다.

- Vue 중심 프론트엔드 포지션 지원에 사용한다.
- 일반적인 이력서의 가독성과 프로젝트 포트폴리오의 설명력을 함께 제공한다.
- 관리자, 입주민, 경비원이라는 서로 다른 사용자의 업무 흐름을 구현한 경험을 강조한다.
- 비공개 소스 코드, 내부 화면, 저장소 링크는 공개하지 않는다.
- GitHub에서 내용과 코드를 관리하고 GitHub Pages로 자동 배포한다.

## 6. 프로젝트 사례 문구

### 6.1 아파트먼트 관리자웹

관리사무소와 최고관리자의 주차, 이사예약, 공지, 문의 업무를 처리하는 Vue 기반 운영 도구다.

주요 기여:

- 기존 입출차 데이터를 기반으로 미출차 내역 화면과 API 흐름 구현
- 미출차 내역에서 입출차 생성, 출차 처리, 삭제 후 이동 흐름과 유효성 검증 개선
- 입출차, 미출차 조회와 삭제 과정에 로딩 및 중복 클릭 방지 상태 적용
- 입출차 동기화의 주차 서비스별 분기 오류 수정
- 재전송 대상이 없을 때 동기화 버튼 상태를 안전하게 제어
- 입출차 내역 삭제와 차량 이미지 표시 기능 개선
- 이사예약의 시간대 중복, 시작 및 종료 시간, 입주 기간, 예약금 조건 검증
- 전체 공지와 문의 기능의 화면, 라우팅, 폼 구조 구현
- 검색 조건 URL 복원, 중복 요청 방지, XSS 및 예외 방어
- 오래된 폼과 Mutation 패턴을 제거하고 공통 버튼과 폼 구조로 리팩토링

### 6.2 아파트먼트 입주민앱

입주민의 출입, 생활, 관리비, 커뮤니티, 예약 업무를 제공하는 모바일 웹뷰 기반 Vue 서비스다.

주요 기여:

- VueUse `useStorage`와 Pinia를 활용한 자동로그인 및 인증 상태 구조 구현
- 로그인, 로그아웃, 토큰 재발급, 보류 요청 상태와 Axios 흐름 개선
- 커머스 인증 토큰 API와 Query를 구현하고 상가 서비스 진입 흐름 연결
- 네이티브 카메라와 연동한 안면인식 등록 흐름 구현
- Base64 이미지 변환, API 상태, 오류 사유, iOS 뒤로가기와 이벤트 정리 처리
- 소방 자가점검의 작성, 서명, 제출, 내역, 상세 전체 흐름 구현
- 관리비 차트와 조회 화면 구현
- 중복 API 호출, 캐시 정책, 목록 스크롤 상태 개선
- 댓글과 답글의 다중 이미지 등록, 수정, 삭제 및 업로드 진행 상태 구현

조식예약 PR은 본인 계정으로 열렸지만 내부 커밋 작성자가 다른 개발자로 기록되어 있으므로 본인 단독 성과로 표시하지 않는다.

### 6.3 아파트먼트 경비원웹

경비원의 차량 출입과 현장 장비 설정 업무를 지원하는 Vue 기반 웹 서비스다.

주요 기여:

- LPR, 입차 정책, 방문증 키오스크 설정 화면과 API 연동
- 미정산 차량, 상가 유형, 출차 게이트 관련 운영 흐름 개선
- 블랙리스트 차량 입차 시 음성 알림 기능 구현
- 원격 음원 오류 시 로컬 음원으로 대체하는 재생 흐름 구성
- 이미지 URL 처리를 공통 컴포저블로 분리
- GitHub Actions 기반 개발 배포와 브라우저 갱신 자동화

## 8. 시각 디자인

### 디자인 읽기

한국의 채용 담당자와 프론트엔드 면접관을 위한 개발자 포트폴리오다. 밝고 신뢰감 있는 에디토리얼 언어를 사용하며, 일반 이력서보다 기억에 남지만 실험적인 에이전시 포트폴리오처럼 보이지 않게 한다.

### 디자인 다이얼

- `DESIGN_VARIANCE: 6`
- `MOTION_INTENSITY: 4`
- `VISUAL_DENSITY: 4`

### 토큰과 규칙

- 기본 색상: 차가운 오프화이트, 차콜, 중성 회색
- 강조 색상: 코발트 블루 한 가지
- 테마: 시스템 설정을 따르는 라이트 및 다크 모드
- 글꼴: 한글 가독성이 높은 산세리프, 기술 표기에 제한적인 모노스페이스
- 모서리: 작은 라운드 규칙으로 통일
- 그림자: 필요한 계층에만 약하게 사용
- 카드보다 여백과 구분선을 우선 사용
- 보라색 글로우, 가짜 터미널, 과한 유리 효과, 3개 동일 카드 배열은 사용하지 않음

### Hero

- 데스크톱은 텍스트와 프로필 비주얼의 비대칭 2열 구성
- 모바일은 한 열로 축소
- 헤드라인은 데스크톱 2줄 이내
- 보조 문구는 짧게 제한
- CTA는 경력 보기와 이메일 두 개 이하
- GitHub 프로필 이미지가 실패하면 조태현의 이니셜 비주얼로 대체

### 모션

- 첫 화면의 텍스트와 이미지 진입
- 주요 섹션의 1회 스크롤 등장
- 버튼 hover 및 active 피드백
- `prefers-reduced-motion`에서는 자동 모션 제거
- 스크롤 이벤트 리스너를 사용하지 않고 Intersection Observer를 사용

### 데이터 흐름

- 경력, 프로젝트, 기술, 학력 정보는 `src/data/resume.js`에 저장한다.
- 섹션 컴포넌트는 이 데이터를 props로 받아 렌더링한다.
- 실행 시 비공개 GitHub API를 호출하지 않는다.
- 외부 데이터는 GitHub 프로필 이미지와 공개 프로필 링크만 사용한다.
- 페이지 내 이동은 Vue Router 없이 앵커 링크로 처리한다.

### 오류와 대체 상태

- 프로필 이미지 로드 실패 시 이니셜 표시
- JavaScript가 지연되어도 핵심 이력 콘텐츠는 정적 HTML에 가깝게 즉시 렌더링
- 외부 링크는 새 탭과 `noopener noreferrer` 적용
- 모션을 지원하지 않거나 줄이도록 설정한 환경에서 정적 상태 제공
- GitHub Pages 하위 경로 배포를 고려해 Vite base 경로를 환경에 맞게 설정

## 10. 반응형 동작

- 768px 미만에서는 모든 2열 및 비대칭 레이아웃을 한 열로 변경한다.

## 11. 접근성과 SEO

- 본문과 버튼은 WCAG AA 수준의 명도 대비를 확보한다.
